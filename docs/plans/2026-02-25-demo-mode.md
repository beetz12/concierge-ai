# Demo Mode Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Add a `DEMO_MODE` flag that lets the app run fully without Supabase or VAPI, using Gemini-simulated calls and in-memory/localStorage-only data, with a visible demo banner.

**Architecture:** Two env vars control demo mode: `DEMO_MODE=true` (backend) and `NEXT_PUBLIC_DEMO_MODE=true` (frontend). Backend skips Supabase plugin and returns mock DB responses from in-memory stores. Frontend bypasses auth, uses localStorage-only server actions, and shows a persistent demo banner. Gemini `simulateCall()` already exists and is used for realistic call transcripts without VAPI. Provider search still uses real Gemini API (cheap, no infrastructure needed).

**Tech Stack:** Next.js 16, Fastify 5, React 19, Tailwind CSS 4, TypeScript 5.9, Google Gemini API

---

## Overview of Changes

### What stays the same in demo mode:
- Gemini provider search (real Google Maps grounding) - this is the core demo value
- Gemini task analysis for direct tasks
- Gemini call simulation (already exists at `simulateCall()`)
- Frontend UI, navigation, and component structure
- AppProvider localStorage state management

### What changes in demo mode:
- **Backend**: Skip Supabase plugin, skip VAPI calls, use Gemini `simulateCall()` instead
- **Backend**: In-memory store for call results (no DB persistence needed)
- **Frontend**: Server actions return mock responses instead of hitting Supabase
- **Frontend**: Auth bypassed (no Supabase auth, auto-authenticated demo user)
- **Frontend**: Middleware allows all routes without auth check
- **Frontend**: Demo banner at top of main layout
- **Frontend**: Real-time subscriptions replaced with polling/simulated updates

```markdown
### Demo Mode

Demo mode allows the app to run with only a Gemini API key — no Supabase, no VAPI, no Twilio needed.

**Activation:**
1. Backend: Set `DEMO_MODE=true` in `apps/api/.env` (or copy `apps/api/.env.demo`)
2. Frontend: Set `NEXT_PUBLIC_DEMO_MODE=true` in `apps/web/.env.local` (or copy `apps/web/.env.demo`)

**What happens in demo mode:**
- **Database**: Skipped entirely. Server actions return mock responses with generated UUIDs. AppProvider localStorage handles all state.
- **Auth**: Bypassed. A demo user is auto-authenticated. Middleware allows all routes.
- **VAPI Calls**: Replaced with Gemini `simulateCall()`. Realistic AI-generated transcripts with 3-8 second delays.
- **Provider Search**: Still uses real Gemini API (the core demo value).
- **Real-time**: Supabase subscriptions disabled. Recommendations auto-generated from local state.
- **UI**: Amber "Demo Mode" banner at top. "Simulated" badges on call transcripts.

**Minimum requirements for demo:** Just `GEMINI_API_KEY` in `apps/api/.env`.
```
